## The three architectural camps

### Camp A ‚ÄĒ Wrap the real CLI in a terminal widget (where fleet lives)

| Product | Tech | Notes |
|---|---|---|
| **fleet** | SwiftUI + SwiftTerm + tmux + Go daemon | Only product using tmux as the session backend. |
| Claudia / opcode | Tauri + portable-pty | Only the *secondary* terminal pane wraps a PTY. Primary chat is rebuilt. |
| Crystal / Nimbalyst | Electron + xterm.js | Crystal deprecated 2026-02. Nimbalyst added an iOS companion app. |
| Anthropic Claude Desktop (Code mode) | undocumented | Embeds a terminal ‚ÄĒ mechanism unknown. Validates the category. |
| VibeTunnel | server-side Ghostty WASM | See Camp C below. |

### Camp B ‚ÄĒ Headless agent + native chat UI (dominant 2026 pattern)

| Product | Spawn pattern | Notes |
|---|---|---|
| **Conductor** ($22M Series A) | Tauri + Claude Agent SDK directly (NOT the CLI) | Every interactive feature reimplemented natively. Hooks not mentioned in docs ‚ÄĒ likely unsupported. xterm.js used only for "Big Terminal" escape hatch. |
| **Crystal / Claudia** | `claude -p --output-format stream-json --dangerously-skip-permissions` per turn, parse JSON, render React | Permission prompts bypassed by hard-coded skip flag. Plan mode not handled. ~22 hand-rolled tool widgets. |
| **Cline / Roo / Continue** | `@anthropic-ai/sdk` directly, custom XML tool parser | ~30K LOC reimplementation: 3,764 LOC agent loop, 24 tool handlers, 1,673 LOC MCP hub, 8 hooks (took ~6 months to ship), shadow-git checkpoints. |
| **Zed Agent Panel** | child `claude` process over **ACP** (Agent Client Protocol) JSON-RPC stdio | Anthropic's TS SDK is ACP-compatible. Cross-vendor standard emerging. |
| **Goose Desktop** | `goosed` Rust daemon + Tauri/Electron clients | Closest architectural parallel to fleet's daemon-and-clients design. |
| Sketch.dev, Cursor Agents, Sourcegraph Amp, Devin, Claw | various | All in this camp. |

### Camp C ‚ÄĒ Server-side terminal emulator (third way)

| Product | Mechanism |
|---|---|
| **VibeTunnel** | Compiles **Ghostty to WASM**, runs it in a Node server. PTY bytes flow into Ghostty; server emits structured `BufferSnapshot` (cell grid + cursor + scrollback) to clients. Real PTY underneath, queryable grid above. |

Plausible **fleet V2 path** if we ever need cross-pane search / link-detection / structured selection without giving up real `claude`.

## Top risks in our SwiftTerm + tmux stack (and mitigations)

| Risk | Source | Mitigation |
|---|---|---|
| Default CoreGraphics renderer rebuilds NSAttributedStrings per-line per-frame; Claude in tmux emits 4‚Äď7k scroll events/sec ‚Üí CPU pin | SwiftTerm issue #486 | Enable Metal renderer (#479) day one; buffer PTY bytes daemon-side and drain at 60Hz via `Timer.scheduledTimer` |
| Resize-narrower causes permanent buffer-reflow corruption (orphan/duplicate lines, no recovery) | SwiftTerm issue #494 | Ship a force-reattach hotkey that destroys+recreates the SwiftTerm view against the same tmux session (tmux grid is source of truth) |
| Weak IME (only partial Korean), no XTMODKEYS / Kitty keyboard, no OSC-52 clipboard | source review | Pin SwiftTerm + carry small fork for hot patches |
| `tmux -CC` control mode incompatible with Claude Code's full-screen mode | Claude Code docs | Skip `-CC`. Use plain `tmux attach`. Set `window-size latest` + `aggressive-resize on` for multi-client sanity. |
| Future Claude release tightens TTY check, kills tmux-spawned panes via `process.exit(1)` | VibeTunnel `claude-patcher.ts:79-92` | Watchpoint. If hit, port their binary-patch. |

**V2 escape hatch:** libghostty / libghostty-spm (Kytos, Fantastty, macterm already shipping with it). Full Swift view widget is 6‚Äď12 months out per Hashimoto.

## Verdict

**Stay the course on terminal-attach + tmux + Go daemon + gRPC + SwiftTerm.** Architecturally distinctive; every other camp pays a parity tax we don't. Build the desktop chrome (diffs, PR comments, suggested actions, native approval overlays via MCP-permission-bridge, JSONL-backed transcript view, token/$ meter) **on top of** the real Claude TUI, not in place of it.
